## 🧪 AI-Chemist's Identity in Education
화학/재료 공학 석사 학위와 61억 건의 산업 데이터를 처리해온 데이터 엔지니어로서[cite: 2, 16], 교육 현장의 **'사고 오류'를 '공정 불량'과 같은 데이터 패턴**으로 접근했습니다.

제조 현장에서 센서 간의 종속성을 규명하여 데이터 파이프라인 전처리 시간을 91% 단축했던 엔지니어링 경험을 바탕으로, 본 프로젝트에서는 학습자의 **지문-선지-정답 간의 인지적 종속성**을 규명합니다. Gemini API의 대규모 컨텍스트 처리 능력을 활용하여, 데이터 엔지니어링의 정밀함을 교육 알고리즘에 이식했습니다.

## 🛠️ Core Technology: Gemini-Powered Forecast Engine



### 1. Twin Sync (사고 패턴 동기화)
사용자의 누적 학습 데이터를 통해 고유한 '사고 모델'을 구축하는 핵심 단계입니다.
* **Deep Context Learning**: Gemini 1.5 Pro의 **Long Context Window**를 활용하여 사용자의 과거 풀이 이력 전체(지문, 선택 선지, 소요 시간 등)를 손실 없이 입력하고 고유한 사고 패턴을 추출합니다.
* **Logical Vectorization**: 수집된 데이터를 바탕으로 학습자가 특히 취약한 논리 구조(예: 이중 부정, 인과 역전)를 벡터 공간에 매핑하여 정량화합니다.
* **Impact**: 단순 정답률을 넘어 '어떤 인지적 단계에서 오류가 발생하는가'를 데이터로 환산합니다.

### 2. Error Forecasting (오답 사전 예보)
실제 문제를 풀기 전, AI 트윈이 학습자의 사고 회로를 시뮬레이션합니다.
* **Gemini Simulation Engine**: Gemini API를 통해 새로운 문항의 논리 구조와 사용자의 트윈 모델 간 유사도를 실시간 연산하여 **실수 확률(%)을 산출**합니다.
* **Predictive Alert**: 분석 결과는 사이버펑크 테마의 실시간 대시보드에 시각화되어 학습자에게 선제적 경고를 제공하며, 이는 실전에서의 심리적 안정감을 유도합니다.

## 🏗️ Multi-Agent Architecture & Engineering Optimization
전담 개발자로서 시스템의 재현성과 효율성을 위해 적용한 **Gemini API 최적화 전략**입니다.

* **Logic Extractor Agent**: Gemini의 구조화된 데이터 추출 능력을 활용하여 영어 지문의 논리 구조를 정형화된 JSON 데이터로 변환합니다.
* **Risk Simulator (Twin Engine)**: 학습자의 오답 페르소나를 장착한 Gemini 에이전트가 문항을 풀이하며 오답에 빠질 가능성을 사전에 검증합니다.
* **Performance Optimization**:
    * [cite_start]**Context Caching**: 반복되는 학습 자료 분석 시 Gemini의 컨텍스트 캐싱 기술을 활용하여 **토큰 낭비를 최소화**하고 응답 속도를 향상시켰습니다. [cite: 15, 16]
    * **JSON Mode Enforcement**: 모든 분석 결과를 JSON 형식으로 고정하여 대시보드 UI 연동의 **재현성과 유지보수성**을 확보했습니다.
